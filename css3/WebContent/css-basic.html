Cascading Style Sheets의 줄임말
웹페이지를 디자인해주는(스타일) 프로그래밍 언어.
통일성 있는 디자인을 구성할 수 있고
유지보수가 용이하다.
[예시] http://www.csszengarden.com/

css 문법

선택자 {속성:속성값;}

선택자{ --- css로 꾸밀 대상(주인공) 속성:속성값; --- 꾸밀 내용과 그 값(:은 =의 뜻, ;은 .의 뜻) 속성:속성값; ... }

css선택자

  1. tag선택자 - tag 그대로
    자동, 강제적 적용
  2. ID선택자 - tag에 id를 주고 그 id값 불러와서 선택자로 이용. (#id값)
    id값은 중복적용 불가하므로 주로 구조적인 목적으로 사용.
  3. class선택자 - tag에 class를 주고 그 class값 불러와서 선택자로 이용. (.class값)
    class값은 중복적용 가능하므로 주로 디자인 목적으로 사용.
  4. 기타 등등... 선택자

css적용법 3가지

1. External CSS - 외부 CSS방식 CSS내용을 *.css형식의 파일로 따로 독립시켜 만들어 놓고 html문서와 연결하여 적용. (head태그 영역 안에 link태그를 활용) 2. Internal CSS - 내부 CSS방식 CSS내용을 html문서의 head태그 영역 안에 적용. (head태그 안에 style태그를 활용) 3. Inline CSS - 태그 내장방식 CSS내용을 html문서의 해당 태그 바로 뒤에 적용. (body태그 안의 특정 태그 뒤에 style속성으로 활용) - 세가지 방식 중 CSS적용 우선순위가 가장 높음. - 통일성 있는 디자인 어렵고 유지보수 곤란, 소스가 길어짐

html5에서 추가된 layout용 tag들

<header> </header> --- 머리말 영역(로고, 로그인, 검색창 등이 위치) <nav> </nav> --- 메뉴 영역 <article> </article> --- 컨텐츠 영역(기사나 블로그처럼 독립적, 완셩형 내용) <section> </section> --- 컨텐츠 영역(주제별 구분) <aside> </aside> --- 컨텐츠 영역(페이지 전체의 주제 이외의 퀵링크 광고 등) <footer> </footer> --- 푸터 영역(저작권, 주소, 연락처, 개인정보처리 방침 등이 위치)
